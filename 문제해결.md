# 서버 연결 문제 해결 가이드

## 문제: 서버에 연결할 수 없습니다

### 해결 방법 1: 의존성 재설치

터미널에서 다음 명령어를 실행하세요:

```bash
cd "/Users/jeonghamin/Desktop/덕픽_개인성향_최종 복사본"

# 가상환경이 있으면 삭제
rm -rf venv

# 가상환경 재생성
python3 -m venv venv

# 가상환경 활성화
source venv/bin/activate

# 의존성 설치
pip install --upgrade pip
pip install -r requirements.txt
```

### 해결 방법 2: 서버실행.command 파일 사용

1. **`서버실행.command`** 파일을 더블클릭하세요
2. 자동으로 가상환경을 생성하고 의존성을 설치합니다
3. 서버가 시작되면 터미널에 표시된 주소로 접속하세요

### 해결 방법 3: 수동 실행

터미널에서:

```bash
cd "/Users/jeonghamin/Desktop/덕픽_개인성향_최종 복사본"

# 가상환경 활성화
source venv/bin/activate

# 서버 실행
python app.py
```

### 해결 방법 4: 포트 확인

다른 프로그램이 포트를 사용 중일 수 있습니다:

```bash
# 포트 8080 사용 확인
lsof -i :8080

# 사용 중인 프로세스 종료
kill -9 [PID]
```

### 해결 방법 5: 환경 변수 확인

`.env` 파일이 있는지 확인하고 다음 내용이 포함되어 있는지 확인하세요:

```
GEMINI_API_KEY=your_key_here
GOOGLE_MAPS_API_KEY=your_key_here
```

### 일반적인 오류

#### "ModuleNotFoundError: No module named 'flask'"
- 가상환경이 활성화되지 않았거나
- 의존성이 설치되지 않았습니다
- 해결: `pip install flask flask-cors`

#### "Address already in use"
- 포트가 이미 사용 중입니다
- 해결: 다른 포트 사용 또는 사용 중인 프로세스 종료

#### "Connection refused"
- 서버가 실행되지 않았습니다
- 해결: 서버를 먼저 실행하세요

### 도움이 필요하신가요?

위 방법으로 해결되지 않으면 터미널의 오류 메시지를 확인하고 알려주세요.
